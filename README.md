# 智能桌面看护机器人 — Desktop Watchdog

基于 **K230 庐山派** + **STM32F103** 的嵌入式智能桌面机器人。

- K230 做"大脑"：摄像头人脸识别、语音唤醒、LLM 对话、TTS 语音合成
- STM32 做"身体"：4 路舵机控制、OLED 表情、LED 灯光、电量检测
- 两者通过 UART 串口通信，协议复用现有指令集，**STM32 固件零改动**

---

## 硬件接口

### 引脚连接

| K230 庐山派 | 方向 | STM32 小狗 | 说明 |
|-------------|------|-----------|------|
| UART TX | → | PB11 (USART3 RX) | K230 发指令给 STM32 |
| UART RX | ← | PB10 (USART3 TX) | K230 收 STM32 状态 |
| GND | — | GND | 必须共地 |

### 串口参数

```
波特率:   115200
数据位:   8
校验位:   无 (None)
停止位:   1
流控:     无 (None)
```

---

## 指令协议

**单字节十六进制指令**，K230 通过 UART 发送，STM32 接收后执行对应动作。

此协议完全复用现有蓝牙/语音指令集，STM32 固件无需修改。

### 完整指令表

| 指令 | 动作 | 表情 | 表情 Mode | 动作 Mode |
|------|------|------|-----------|-----------|
| `0x29` | 放松趴下 | 睡觉 | 0 | 0 |
| `0x30` | 蹲下 | 瞪眼 | 1 | 1 |
| `0x31` | 直立 | 眼睛 | 5 | 2 |
| `0x32` | 趴下 | 瞪眼 | 1 | 3 |
| `0x33` | 前进 | 快乐 | 2 | 4 |
| `0x34` | 后退 | 快乐 | 2 | 5 |
| `0x35` | 左转 | 快乐 | 2 | 6 |
| `0x36` | 右转 | 快乐 | 2 | 7 |
| `0x37` | 摇摆 | 非常快乐 | 4 | 8 |
| `0x38` | 步行加速 | — | — | — |
| `0x39` | 摇摆加速 | — | — | — |
| `0x40` | 摇尾巴 (开/关) | 瞪眼 | 1 | 9 |
| `0x41` | 向前跳 | 快乐 | 2 | 10 |
| `0x42` | 向后跳 | 快乐 | 2 | 11 |
| `0x43` | 打招呼 | 打招呼 | 6 | 13 |
| `0x44` | 开启灯光 | — | — | — |
| `0x45` | 关闭灯光 | — | — | — |
| `0x46` | 开启呼吸灯 | — | — | — |
| `0x47` | 关闭呼吸灯 | — | — | — |
| `0x48` | 伸懒腰 | 打招呼 | 6 | 14 |
| `0x49` | 拉伸腿 | 打招呼 | 6 | 15 |
| `0x50` | 显示电量 (开/关) | — | — | — |

### 发送示例（Python）

```python
import serial
ser = serial.Serial('/dev/ttyS0', 115200, timeout=0.1)
ser.write(b'\x43')  # 打招呼
ser.write(b'\x44')  # 开灯
ser.write(b'\x29')  # 睡觉
```

## K230 内部架构

```
main.py (主线程)
├── uart_bridge.py     ← 串口通信，线程安全，全局单例
├── vision/
│   ├── face_detect.py     ← 人脸检测 (NPU kmodel 推理)
│   ├── face_recognize.py  ← 人脸特征比对 (CPU)
│   └── watchdog.py        ← 看门狗状态机
├── chat/
│   ├── voice_wake.py      ← 语音唤醒词
│   ├── llm_client.py      ← LLM API
│   ├── tts.py             ← TTS 语音合成
│   └── emotion_map.py     ← 情绪→动作映射
└── config.py              ← 全局配置
```

### K230 内部接口规范

**所有需要控制小狗的模块，必须通过 `uart_bridge` 发送指令，禁止直接操作串口。**

```python
from uart_bridge import send_command

send_command(0x43)  # 正确：通过统一接口
# ser.write(b'\x43') # 错误：严禁直接操作串口
```

`uart_bridge` 内部实现线程安全锁，多线程并发调用不会导致串口数据错乱。

## 情绪-动作映射（参考）

| 触发条件 | 小狗动作 | 指令序列 |
|----------|---------|---------|
| 识别到熟人 | 打招呼 → 摇尾巴 | `0x43` → `0x40` |
| 检测到陌生人 | 后退 + 瞪眼 + 灯光闪烁 | `0x34` → `0x44` → `0x45` → `0x44` |
| 无人超时 | 放松趴下 + 关灯 | `0x29` → `0x45` |
| LLM 对话：开心 | 摇摆 + 非常快乐表情 | `0x37` |
| LLM 对话：生气 | 蹲下 + 瞪眼 | `0x30` |

## 环境要求

### STM32 (人 A)
- Keil MDK-ARM (或替代 IDE)
- FlyMcu / CS32 ISP Programmer (烧录工具)
- CH340 USB 转串口驱动

### K230 (人 B + 人 C)
- CanMV K230 SDK
- Python 3.x
- `pyserial`
- 嘉楠 nncase (仅当需要自定义 kmodel 时)
- SDK 自带 kmodel: `face_detection.kmodel`, `face_recognition.kmodel`

---

## 快速开始

1. **硬件连接**: 按上方接线表连接 K230 和 STM32
2. **烧录 STM32**: 用 FlyMcu 烧录 `stm32/firmware/Project.hex`
3. **部署 K230 代码**: 将 `k230/` 目录拷贝到 K230 开发板的 SD 卡
4. **运行**: `python3 main.py`
