# 智能桌面看护机器人 — 开题报告大纲

## 基于 K230 庐山派与 STM32 的嵌入式 AI 系统设计

---

### 第1页 | 封面

- 课题名称：智能桌面看护机器人
- 副标题：基于 K230 庐山派与 STM32 的嵌入式 AI 系统设计
- 课题类型：嵌入式系统课程设计
- 团队：三人

---

### 第2页 | 研究背景

**核心钩子**：传统电子宠物只是一个"会动的玩具"，缺乏感知能力和交互智能。

- 市面电子宠物功能单一：预设动作 + 遥控，无环境感知
- AI 边缘计算芯片（K230）的成熟让本地 AI 成为可能
- 大语言模型的普及使机器人可以真正"对话"
- 将 AI 能力注入嵌入式硬件，是嵌入式系统课程的前沿方向

**一句话**：给机器狗装上眼睛、耳朵和大脑。

---

### 第3页 | 方案对比

| 传统电子桌宠 | 本课题方案 |
|-------------|-----------|
| 蓝牙/红外遥控 | 人脸识别自动感知 |
| 预设动作序列 | AI 情绪驱动动作 |
| 无交互或简单语音指令 | LLM 自然语言对话 |
| 单一 MCU | K230(大脑) + STM32(身体) 异构双核 |
| 玩具属性 | 实用看护属性 |

---

### 第4页 | 系统总体架构

三层架构设计：

```
感知层 (K230 NPU)
├── MIPI 摄像头 → 人脸检测与识别
├── I2S 麦克风 → 语音唤醒 + STT
└── WiFi → 云端 LLM API

计算与控制层 (K230 CPU + STM32)
├── K230: 主控逻辑、情绪映射、动作决策
├── UART 串口通信 (115200 8N1)
└── STM32: 实时舵机控制、OLED 显示

执行层 (STM32 外设)
├── 4 路舵机 → 行走/跳跃/摇摆/摇尾巴
├── OLED 128×64 → 6 种表情
├── LED + 呼吸灯
└── 电池电量检测
```

### 第5页 | 硬件平台

**K230 庐山派（大脑）**
- 算力：1.6 TOPS NPU
- 摄像头：MIPI CSI 接口
- 通信：WiFi + UART
- 系统：Linux + Python

**STM32F103 电子小狗（身体）**
- 舵机 ×4 + 尾巴舵机 ×1
- OLED 128×64 显示屏
- LED 灯光系统 + 呼吸灯
- 电池供电 + 电量检测

### 第7页 | 四大核心功能

1. **人脸识别看护** — NPU 本地推理，熟人欢迎、陌生人告警
2. **LLM 智能对话** — WiFi 连接云端大模型，自然语言聊天
3. **情绪动作映射** — LLM 情绪 → 小狗动作（快乐→摇摆，警觉→后退）
4. **语音交互** — 唤醒词检测 + TTS 语音合成

---

### 第8页 | 技术实现路线

摄像头采集 → NPU 人脸检测 → 特征比对 → 身份判定 → 
唤醒词检测 → STT 语音识别 → LLM 对话 → 情绪提取 → 
动作映射 → UART 指令 → STM32 执行 → 舵机 + OLED 响应

共 10 个步骤，形成完整的感知-决策-执行闭环。

---

### 第9页 | 课题难点与创新点

**三大难点**
1. **异构双核通信** — K230 Linux 与 STM32 裸机的 UART 协议设计与同步
2. **NPU 模型部署** — .kmodel 的量化、编译与实时推理优化
3. **多模态融合** — 视觉、语音、动作三种模态的时序协调

**创新点**
1. 边缘 AI + 传统 MCU 的异构架构
2. LLM 情绪到物理动作的映射机制
3. 本地视觉推理 + 云端语言推理的混合智能
